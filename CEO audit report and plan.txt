2. CEO VISUAL AUDIT FINDINGS
==================================================

2.1 Asset section issue ✅ DONE
- Current issue: there is no option to delete an asset.
- Current visible actions: only download and preview.
- Business problem: if someone mistakenly uploads something, they currently have no self-service way to delete it.
- Required future action: add asset deletion capability.
- Required UX consideration:
  - show a clear delete action ✅
  - include confirmation before delete ✅
  - clarify whether delete is permanent or recoverable ✅
  - define role permissions for client delete vs admin delete ✅
- Required backend consideration:
  - remove file record from database ✅
  - remove storage object from Supabase Storage ✅
  - keep audit visibility if needed for internal ops ✅
- IMPLEMENTED: Trash2 icon, inline Confirm/Cancel dialog, DELETE /api/portal/assets/[assetId] with auth + ownership check, Storage + DB cleanup

2.2 Product selection section needs major content expansion
- Current issue: the product selection section needs more products.
- Current issue: products need images.
- Current issue: additional assets and richer product information are needed.
- CEO note: updated versions may need to be shared section by section.
- Planning requirement: define a structured content intake workflow so updates can be shared clearly and repeatedly without confusion.

2.3 Recommended workflow for sharing updated product and section assets
- Best practice for now: create one structured source file per section update.
- Recommended file formats:
  - `.txt` if plain structured notes are enough
  - `.md` if sectioning, links, and checklists are helpful
  - `.csv` if bulk product catalog entries are being added
- Recommended content structure for each section update:
  - Section name
  - Page name
  - Item name
  - SKU
  - Category
  - Short description
  - Long description
  - MOQ
  - Material
  - Decoration compatibility
  - Production benchmark
  - Pricing benchmark
  - Image filename or image path
  - Priority level
  - Replace existing item or add as new item
  - Notes for UX placement
- Recommended operational rule:
  - one source-of-truth file per section
  - one image folder per section
  - consistent naming for all uploaded image assets
- Recommended naming examples:
  - `catalogue-products-update-v1.txt`
  - `quote-product-selection-update-v1.txt`
  - `history-section-reference-notes.txt`
  - `account-section-edit-flow-notes.txt`
- Future best workflow:
  - maintain section update files in a dedicated `/docs/content-updates/` directory
  - maintain product images in a dedicated `/public/images/catalogue/` or section-specific image folder
  - connect future admin CRUD tools so manual source files become optional

2.4 Future admin panel editing capability is required
- CEO requirement: in the future, admins must be able to add, remove, and edit the whole product catalogue.
- This includes:
  - product name
  - images
  - descriptions
  - categories
  - materials
  - pricing ranges
  - MOQs
  - logistics metadata
  - decoration methods
  - all supporting product data
- Required system direction:
  - admin catalog management must become database-driven
  - admin panel must support CRUD operations
  - uploaded product images must be manageable
  - validation rules must prevent broken catalogue entries

2.5 Catalogue section needs stronger assets and modern functionality

- Observed planning need:

  - richer filters
  - stronger product cards

- Future functionality ideas:
